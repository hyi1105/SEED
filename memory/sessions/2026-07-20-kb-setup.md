# 建立 AI-first 知識庫運作方式

## 摘要

使用者要把與 Cursor Agent 的對話變成可檢索記憶，並**一起維護筆記**：Agent 直接改 `.md`，使用者用 Cursor 看 diff 確認（像審 code）。平常不靠自己摸索檔案；需要回憶時由 Agent 撈出並說明日期與內容。貼圖要存進庫並轉成文字記憶。目前**只有公司電腦有 Cursor、沒有第二台電腦**；關機後希望手機繼續。

## 重點

- 格式：Markdown + 索引；筆記寫到人掃一眼也看得懂，避免只有 Agent 懂的縮寫。
- 協作方式：Agent 改檔 → 使用者在 Cursor 確認變更（diff）→ 像寫 code 一樣審核。
- 回憶：問過去內容時，Agent 回報日期時間與摘要；必要時可重整成主題筆記。
- 貼圖：存 `memory/assets/` + 筆記內文字記憶（畫面說明 + OCR）。
- 跨裝置：庫在 OneDrive；以文字記憶為主方便手機讀。

## 決定／偏好

- **介面**：以 Cursor 對話 + diff 審核為主；不是「自己打開陌生 md 硬找」。
- **寫入方式**：有價值內容由 Agent 直接改筆記，並在聊天裡簡短說明動了哪裡。
- **圖片**：原圖 + 文字記憶並存；文字為可搜尋主體。
- **手機（硬限制）**：公司電腦**關機**後，遠端遙控不可用（需要電腦開著）。要在手機「繼續跟 Agent 聊並改筆記」，需走 **雲端 Agent**（iOS 應用或瀏覽器 `cursor.com/agents`），且知識庫最好變成雲端可存取的 Git 倉庫；公司資安／帳號權限可能擋這條路。
- **手機（讀取）**：OneDrive 應用可讀已同步的文字筆記；這是關機後最低成本方案。
- **裝置現況**：僅公司電腦有 Cursor，無其他電腦。
