# 減少 UI 來回：溝通方式共識

## 摘要

使用者覺得畫面一直重複調整浪費時間，詢問更好的溝通方式，以及高手怎麼處理。

## 重點

- 來回成本高的主因是目標未鎖死，不是單純實作慢
- 建議：brief（目標／參考／必做／不要／完成定義）→ 再動手
- 已整理成主題筆記供之後沿用

## 決定／偏好

- 大改畫面時，優先用「像 XX」＋完成定義，少用模糊形容詞

## 待續

- 之後 UI 任務可先問一句：要不要先寫五句 brief 再改？
