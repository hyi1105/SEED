# 目標會變時怎麼快速大改

## 摘要

使用者指出：常是看過成品才想到更好做法，或對成品有新需求，目標本來就會變。重點不是「先寫死 brief」，而是把探索與定稿分開，轉向時用留／丟／新做一次大改。

## 重點

- 看了才知道要什麼＝正常學習迴路
- 貴的是把探索版當定稿一直微調
- 快速大改句型：留／丟／新＋這一輪完成定義

## 決定／偏好

- 採用「探索 → 轉向大改 → 定稿」；大改不在舊殼補丁

## 待續

- 之後 UI 任務若使用者沒標模式，Agent 可先問一句：這輪探索還是定稿？
