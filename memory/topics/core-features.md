# SEED 核心功能清單（不可少）

> 畫面不必 100% 一樣；**下列能力不可缺**。實作可換皮、換技術，但體驗要在。

## A. 知識拼圖（首頁）

| 能力 | 說明 |
|---|---|
| 格子拼圖 | 可設定大小（模板 8／10／12…），無東南西北 |
| Seed 上圖 | 每個 `.md` 可擺在一格 |
| 拖曳改位 | 可拖到空格；可與他格交換 |
| 章印／圖示 | 地圖標記可辨識（現為圖示＋短標，可換書封 cover） |
| 點開讀取 | 點一格 → 讀該 seed 正文 |
| 位置持久 | 可寫回倉庫（`seeds.json` 的 col/row） |
| 模板可選 | 空白格／示範模板等（見 `knowledge-puzzle.md`） |

## B. 煉化／版本（阿嬤六動作）

| 能力 | 說明 | 現況 |
|---|---|---|
| 自己改 | 人手改正文 | **已有** |
| AI 回答 | System 先說、User 回答、AI 再回應 | **已有**（自備 key）；**付費路徑**：平台代辦 key（見 `paid-grandma-ai.md`） |
| 看看改了什麼 | 舊／新對照（紅綠或等價） | **已有**（草稿差異＋Git 歷史） |
| 用這次的／不要 | 接受或丟棄建議 | **已有**（草稿） |
| 存成一版 | 文件快照，可取名 | **已有**（寫回 `.md`） |
| 回到舊的 | 選一版還原／預覽 | **已有**（可「用這版繼續改」） |
| 分享／打包 | 可帶走、可還原的完整封包 | **已有**（打包帶走／還原回來） |

## C. 內容與協作原則

| 能力 | 說明 |
|---|---|
| Markdown 為本 | 筆記是 `.md`，人可掃、AI 可改 |
| 只放可公開學習筆記 | Public 策略；敏感資料不進庫 |
| 用語藏工程 | 介面不說 commit／push／branch |
| 與 Cursor 共維 | Agent 改筆記 → 人看 diff 確認 |

## D. 刻意不算核心（可晚做）

- 多來源自動抓取、法務判決即時、多人即時協作、登入授權、手機原生應用、真經緯度地圖

## 判定標準

重做或換實作時：若 **A 全有**，且 **B** 除「請 AI 改／完整 seed 包分享」外皆可用，即核心不缺。畫面長相不列入判定。
