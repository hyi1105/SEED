# SEED 封包：打包帶走／還原回來

## 摘要

個人全部內容與設定要能打成一個可攜帶檔（概念上像 PDF：帶走、跨平台），在別的 SEED／裝置上傳後可恢復。

## 阿嬤用語

| 按鈕 | 意思 |
|---|---|
| **打包帶走** | 下載一份完整備份 |
| **還原回來** | 選檔上傳，寫回倉庫 |

## 封包裡有什麼

- 格式：`seed-pack`（副檔名建議 `.seedpack.json`）
- 地圖設定、每個 seed 的位置／短標／路徑
- 各筆記 `.md` 全文（以及種子清單 `docs/seeds.json`）
- 瀏覽器地圖擺位（若有）
- **不含** GitHub 鑰匙、AI 鑰匙

## 不含／限制

- 不是真的 PDF（PDF 難完整還原編輯）；封包是可還原的資料檔
- Git 歷史每一版不會整包帶走（帶走的是「現在這份」全文）；歷史仍在原 GitHub
- 還原需要目標環境已設 GitHub 鑰匙

## 使用節奏

1. 換電腦前：打包帶走 → 存到雲碟／隨身碟  
2. 新地方開 SEED → 設定鑰匙 → 還原回來  
3. 公司電腦可再 `git pull` 同步 OneDrive 工作區
