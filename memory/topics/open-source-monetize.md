# 公開 code／想法時怎麼賺錢（AI 容易抄的年代）

## 一句話

**公開的是可被複製的「配方」；收費的是難複製的「持續交付」**——信任、分發、託管、資料、社群、服務、速度。

## 為什麼「怕被 AI 抄」是對的、但不全對

| 容易被抄 | 難被抄（或抄了也賺不到你的錢） |
|---|---|
| README 構想、UI 草圖、演算法敘述 | 你的真實使用場景與反饋循環 |
| 開源 repo 的程式碼 | 已上線、有用戶、有口碑的產品 |
| 一次性 Demo | 持續更新、支援、合規、整合 |
| 「阿嬤級 UX」這句定位 | 做出來、教別人用、幫人落地的你 |

AI 降低的是**從 0 到能跑**的成本，不是**從能跑到有人付費**的成本。

## 可並行的賺法（由易到穩）

### 1. Open Core（對 SEED 最貼）

- **公開**：個人學習用、本機／Pages 版、核心六動作
- **收費**：雲端同步、多人協作、權限、備份、搜尋、組織版、SLA
- 抄你的人仍要自己扛伺服器、帳號、客服、信任——多數抄完就停

### 2. 服務與時間（最快有現金）

- 幫公司／團隊導入「可煉化筆記」流程（工作坊、建庫、交接包）
- Cursor／知識庫顧問：你已在做的事，包裝成可報價項目
- 不賣「想法」，賣「兩週內幫你把流程跑起來」

### 3. 內容與分發（想法公開的真正用途）

- 公開是**獲客**：教學、案例、seed 範本庫
- 收費在後面：進階課、模板包、社群、一對一
- 想法愈公開，搜尋與口碑愈容易找到你——抄襲者沒有你的頻道與關係

### 4. 資料與網路效應（中期護城河）

- 公開 code，但**優質 seed／地圖／場景模板**累積在你這邊
- 使用者產出的 refinement 路徑、好範本、評分——這才是產品變厚的部分
- AI 能抄界面，抄不走「一堆真實用過的種子」

### 5. 授權與品牌（選用）

- 開源用寬鬆授權（MIT）換採用量；商業功能另授權
- 或 AGPL／雙授權：想閉源商用就付錢
- 對個人品牌：公開 = 作品集 + 信任狀，比保密構想更值錢

## 對 SEED 的建議順序

1. **現在**：公開個人 MVP + 學習筆記（已定）→ 當作品與獲客，不急著收費
2. **有真實第二個使用者後**：加一層「託管／同步／備份」當第一個付費點
3. **並行**：把「幫人建可煉化知識庫」做成固定報價服務（現金流）
4. **不要**：指望靠保密構想或單純賣程式碼檔；那條路已被 AI 壓扁

## 決策檢查（每次想公開前問自己）

- 這份公開內容，主要是為了：**獲客／信任／協作**，還是誤以為會「被偷走就沒了」？
- 我收費的單位是什麼：**主機位／時間／結果／訂閱**，還是「檔案本身」？
- 抄襲者要追上我，還缺哪三樣：**用戶、資料、更新節奏**？
