# 怎麼跟 Agent 講 UI，少重複改

## 為什麼會一直改

畫面問題多半不是「寫錯」，而是**目標沒先鎖死**：像什麼、不像什麼、做到哪算完。沒鎖死就只能用成品猜，來回最貴。

## 世界上強的人怎麼做（濃縮）

1. **先寫 brief，再動手**  
   產品／設計強者先把「誰、做什麼、成功長怎樣」寫成半頁，不是邊做邊發明。
2. **用參考，少用形容詞**  
   「直覺一點」成本高；「像 LINE 加表情：下方托盤拖進內容」一次講準。
3. **一次只改一層**  
   先結構 → 再互動 → 再視覺。同一輪不要混「分類架構＋拖移手感＋配色」。
4. **先看方向，再看細節**  
   草圖／截圖／diff 先點頭，再打磨像素。不在第一版挑邊距。
5. **寫「不要什麼」**  
   排除項比願望清單更能減少做錯方向（例如：不要表單列、不要↑↓按鈕）。
6. **把偏好存成系統**  
   重複出現的品味寫進筆記／設計規則，下次不用重講。

## 跟本庫 Agent 建議用的句型

開做前貼這五句（可删減）：

```text
1. 目標：使用者在這個畫面要完成____
2. 參考：像____（App／先前功能／截圖）
3. 必做：____
4. 不要：____
5. 完成定義：做到____就停，細節下一輪再說
```

UI 回饋時改成「診斷＋處方」：

```text
現在問題：____（我卡在哪）
我想要的感覺：像____
請只改：____（其他別動）
```

## 本專案已驗證的好例子

- 差：「編輯模板不直觀」→ 容易猜錯成換按鈕文案  
- 好：「元件用拖移，如同新增表情符號」→ 一次對到托盤拖放

## 決定／偏好

- 大改 UI 前，優先要 brief／參考／完成定義，而不是先出一版再慢慢調
- 使用者可用「像 XX」當最高槓桿溝通方式
