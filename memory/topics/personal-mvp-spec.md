# 個人 MVP 規格（一頁）

## 目標

做出「80 歲阿嬤也會用」的**最小可煉化文件**：自己或請 AI 改 → 看改了什麼 → 用／不要 → 存一版 → 可回舊版 → 可分享。  
第一真實場景：**學 Cursor／新系統時，把過程煉成一份可交接的筆記。**

非目標（第一版不做）：多來源連接器、法務判決、團隊權限、手機原生 App。  
**託管**：公開 repo [`hyi1105/SEED`](https://github.com/hyi1105/SEED)；操作畫面目標做在 **GitHub Pages**（Public）。只放可公開學習筆記。

## 使用者（第一版就是你）

- 會打字、會點按鈕
- 不懂 Git／GitHub 用語
- 要邊學邊留一份「愈改愈清楚」的文件

## 六個動作（唯一主流程）

| # | 阿嬤看到的 | 系統做的事 |
|---|---|---|
| 1 | **自己改** | 直接編輯正文 |
| 2 | **請 AI 改** | 依一句話改正文（先出「建議變更」，未寫入前可預覽） |
| 3 | **看看改了什麼** | 對照舊／新（顏色標出增刪） |
| 4 | **用這次的／不要** | 接受或丟棄這次 AI／編輯建議 |
| 5 | **存成一版** | 存快照；可取名（預設：時間，如「7月20日 早上」） |
| 6 | **回到舊的** | 選一版還原成目前正文（還原前可再存一版以免後悔） |
| — | **分享這一份** | 匯出一個 seed 包（見下） |

## 畫面（首頁＝知識地圖）

```text
        北
   ┌──────────────────┐
西 │ 10×10 格子地圖    │ 東
   │ 種子可拖到空格    │
   └──────────────────┘
        南
左下＝工作室筆記；中央＝地理示範（北中南橫／蘇花）
點一格 → 讀／看差異／回舊版
```

版本列表（「回到舊的」打開）：每列＝名稱 + 時間；點一下可預覽，再確認「用這版」。

## 資料（自建版本，簡單即可）

每份文件一個資料夾（或一筆文件 ID）：

- `current.md` — 現在這份正文  
- `versions/` — 每一版一份檔（或一列 DB）：`id、名稱、時間、全文`  
- `meta.json` — 標題、建立時間、場景模板名  

第一版可用**本機檔案**，不必上雲。

## Seed（分享單位）

「分享這一份」= 打成一個資料夾／壓縮包，內含：

- 目前正文  
- 版本歷史（可選：第一版先只帶目前正文 + 最後 3 版，以免太大）  
- 一句說明：這是什麼、給誰看  

對方之後可「開這一份」繼續煉（同一套六動作）。

## 場景模板：學新系統／Cursor

新建文件時可套用標題與大綱（可改）：

```markdown
# （系統或專案名）上手筆記

## 我現在要做什麼
## 我已經會的
## 今天學會的
## 常踩的坑
## 重要決定
## 還不懂、下次要問的
## 給接手的人（三句話）
```

AI 改的時候，預設提示方向：把對話裡的學習重點**寫進對應小節**，不要長篇廢話；專有名詞旁加一句白話。

## 接受標準（你自己用一週算過關）

- 不用任何 Git 字就能完成六動作  
- 至少存過 5 版、回退過 1 次、分享過 1 次（匯出）  
- 用「學 Cursor」模板真實寫完一篇可給別人看的筆記  

## 刻意不做

- 登入／多人同時編輯  
- Excel／網頁自動抓取  
- 漂亮版面以外的裝飾  
- 與 Cursor IDE 深度外掛（第一版是獨立小工具或本機 Web 亦可）

## 下一步實作順序

1. 本機單頁：開檔、編輯、存版、列表、回退  
2. 接上「請 AI 改」+ 看看改了什麼 + 用／不要  
3. 套用「學新系統」模板  
4. 分享＝匯出 seed 包
