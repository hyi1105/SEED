# 收費決策：阿嬤只付費，其餘代辦

## Summary

阿嬤不會去申請 AI key（也不會搞 GitHub PAT）。產品要變成：**使用者只負責付錢**；AI 改、鑰匙、（之後）託管與還原等由平台代辦；營運方從中收取**服務費／差額**。

## 產品一句話

**免費可學、可看、可自己改；「請 AI 改」等代辦能力＝付費。**

## 誰付什麼

| 角色 | 做什麼 |
|---|---|
| 阿嬤／付費用戶 | 註冊、綁卡或月費、按「請 AI 改」 |
| 你（營運） | 準備模型額度、代理呼叫、配額、客服、帳單 |
| 公開／自架版 | 仍可自備 key（給會搞的人）；不當預設路徑 |

## 收費單位（建議先選一個）

1. **月訂閱**（最阿嬤）：每月固定額，含 N 次「請 AI 改」  
2. **點數包**：買點數，每次 AI 改扣點（彈性，但多一步）  
3. **兩者並用**：訂閱含基本額度，用超再買點數  

第一版建議：**月訂閱 + 次數上限**（畫面只說「本月還能請 AI 改幾次」）。

## 技術必要條件（相對現況）

現在：瀏覽器直接打 OpenAI＝使用者自己貼 key。  
要收費代辦，至少要：

1. **後端代理**：瀏覽器 → 你的伺服器 → 模型（key 永不進前端）  
2. **帳號／登入**（可先 Email + magic link）  
3. **帳單**：Stripe／綠界等 → 訂閱狀態寫進你的帳號  
4. **配額**：每次 AI 改檢查「還有沒有次數」  
5. **免費層**：自己改／讀／地圖／打包仍可用；按鈕「請 AI 改」引導付費或試用

GitHub 寫回倉庫也可後期代辦（「一鍵存到雲端」），但**第一刀收費點＝代辦 AI**，因為痛點最明顯。

## 定價直覺（待驗證，非定案）

- 試用：新帳號送少量 AI 次數  
- 個人月費：覆蓋模型成本 × 安全倍數 + 服務利潤  
- 成本透明記帳：每次呼叫記 token／金額，避免訂閱被刷爆
