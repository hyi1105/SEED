# SEED

公開的知識煉化實驗：把學習筆記變成可分享、可追蹤變化的 **seed**。

- Repo：https://github.com/hyi1105/SEED  
- 原則：**只放可公開的學習筆記**，不放公司敏感資料  
- 目標：不認識的人也能看、能追知識怎麼累積與改寫；操作畫面做在 **GitHub Pages**（阿嬤也會用的用詞：存一版、看看改了什麼、回到舊的）

## 路線

| 階段 | 內容 |
|---|---|
| 現在 | Markdown 知識庫公開在 GitHub，隨時可看歷史 commit |
| 下一步 | GitHub Pages 操作畫面（六動作 UX） |
| 之後 | 版本差異可視化、seed 分享給陌生人 |

## 維護方式

在 Cursor 與 Agent 共同改筆記 → 看 diff 確認 → push 到本 repo。  
Pages 上的「阿嬤介面」會包住這些能力，不露出 Git 術語。
